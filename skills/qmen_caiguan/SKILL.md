# 财官诊断 — 财富事业双维度深度分析

> 基于奇门遁甲起局，从财富和事业两个维度进行七要害诊断，定位六害（刑墓庚虎迫空），给出化解方向和"踩一捧一"建议。

## Trigger

当用户提及以下内容时激活本技能：
- 财官诊断、财富分析、事业分析
- 看财运（深度）、事业诊断、七要害
- "帮我看看财运和事业"、"分析一下我的局"
- 任何同时涉及财富+事业方向的奇门占断

**注意**：如果用户只是简单问"帮我算一卦"，优先使用 `qmen_analysis`（通用解盘）。本技能用于需要深度财富/事业诊断的场景。如果用户需要布阵/灭象/风水布局，使用 `qmen_huaqizhen`（化气阵）。

## 对话流程

### Step 1: 收集信息

向用户询问信息：

**必填：**
1. **起局时间** — 格式 `YYYY-MM-DD HH:MM`。如果用户说"现在"或不指定，使用当前时间。
2. **出生时间** — 格式 `YYYY-MM-DD HH:MM`，完整的年月日时。必须提供完整出生时间，无默认值。需要提取出生年的天干（用于计算生年财）。如果用户只提供年份，按下表推算天干：

| 年份尾数 | 天干 |
|---------|------|
| 4 | 甲 |
| 5 | 乙 |
| 6 | 丙 |
| 7 | 丁 |
| 8 | 戊 |
| 9 | 己 |
| 0 | 庚 |
| 1 | 辛 |
| 2 | 壬 |
| 3 | 癸 |

示例对话：
```
AI: 财官诊断需要以下信息：
    1. 起局时间（具体时间，或"现在"）
    2. 出生时间（完整的年月日时，例如：1973-04-24 19:30）
```

### Step 2: 封局提醒

如果用户已有未封的局，先提醒封局：起局的纸盖雷霆都司印后封存。动念封局，心中想一位中国英雄（如岳飞、关羽、霍去病、杨靖宇、钱学森、邓稼先），然后删掉遁甲局。

如果是首次起局，跳过此步。

### Step 3: 起局

起局前，**必须向用户展示入局祝福**（任选古版或现代版）：

古版：
> 天门天门，急来开之。
> 我欲为某事，要往殿廷。
> 速为我抽取金关，为我速收金钥。
> 救我形身，辅我勿停。

现代版：
> 所有符号，听我号令。
> 世间万象，助我性命。

展示祝福后，**必须向用户展示结语**：

> 临兵斗者皆阵列前行。

然后**必须执行**以下起局脚本生成命盘和事件盘。**所有命令必须设置 `workdir` 为技能目录。**

```bash
# workdir: {SKILL_DIR}
# 生成命盘（出生时间）
bin/qimen.sh --type=birth "1973-04-24 19:30"
# 生成事件盘（起局时间）
bin/qimen.sh "2024-03-15 14:30"
```

**脚本输出规则：将脚本的全部 stdout 输出原样粘贴展示给用户。禁止截断、省略、总结或改写任何一行输出。**

### Step 4: 财官分析

**必须执行**财官分析脚本（自动从 `./qmen_birth.json` 读取出生年天干，行业从盘上自动推算）：

```bash
# workdir: {SKILL_DIR}
bin/qimen_caiguan.sh
```

**脚本输出规则：将脚本的全部 stdout 输出原样粘贴展示给用户。禁止截断、省略、总结或改写任何一行输出。** 脚本默认读取 `./qmen_birth.json`（命盘），写入 `./qmen_caiguan.json`。如需分析事件盘，使用 `--input=./qmen_event.json`。

### Step 5: 解读

**必须先完成 Step 3 和 Step 4 的脚本执行**，然后用 Read 工具读取 `./qmen_caiguan.json`，按照「解读框架」生成中文叙事分析。

```
Read: {SKILL_DIR}/qmen_caiguan.json
```

### Step 6: 追问处理

解读完成后告诉用户可以继续提问。如果用户需要布阵化解，引导到 `qmen_huaqizhen` 技能。

## 起局仪轨

### 不起局条件

以下情况不应起局：
- **小事** — 无关紧要的琐事
- **心乱** — 心神不宁，念头不纯一
- **害人** — 目的为害他人

### 起局条件

- 利己、利人
- 心念纯一，绝境求生
- **一事一局** — 事无变化，不起新局

## 核心概念

### 六害

六害是盘中对符号的六种危害因素：

| 六害 | JSON 标记 | 含义 |
|------|----------|------|
| 刑 | 刑 | 六仪击刑，受刑伤、纷争 |
| 墓 | 干墓/星墓/门墓 | 入墓，被困、陷入泥沼 |
| 庚 | 庚 | 天盘见庚，大阻力大障碍 |
| 庚(对宫) | 庚(对宫) | 对宫有庚，间接阻力 |
| 虎 | 虎 | 白虎同宫，凶猛压力 |
| 虎(对宫) | 虎(对宫) | 对宫有白虎，间接压力 |
| 玄武(对宫) | 玄武(对宫) | 对宫有玄武，暗中影响 |
| 迫 | 迫 | 门迫，通道不畅 |
| 空 | 空 | 空亡，落空虚幻 |

**六害叠加越多越严重。** 单个六害是警告，两个以上是危险信号。

### 财富与事业的关系

- **财富**：掌控的资源，货币化为金钱。优先考虑自己。
- **事业**：社会角色和伴生权利。优先考虑别人。
- **二者不可兼得**：需要"踩一捧一"，即在某一维度让步以保全另一维度。强行两手抓必出问题。

### 月令五行关系

月令是起局时月支的五行，代表大环境和天时。JSON 中 `yuegling.palaces[].relation` 字段已计算好，`yuegling_relation_cn` 字段提供中文含义：

| relation 值 | yuegling_relation_cn | 含义 | 对财富/事业的影响 |
|------------|---------------------|------|-----------------|
| sheng | 扩张量大 | 月令生此宫 | 扩张，量大，事半功倍 |
| tong | 稳健量大 | 月令同此宫 | 稳健，量大，根基牢固 |
| ke_target | 大亏量小 | 月令克此宫 | 大亏，量小，最差 |
| sheng_target | 损耗量小 | 此宫生月令（泄耗） | 损耗，量小 |
| bei_ke | 努力可成 | 此宫克月令 | 努力可成，量小 |

### 天干五合

局中找不到需要的天干时，找其所合的天干替代。JSON 中 `gan_cai.stems[].wuhe_alt` 已标注替代天干。

## 解读框架

读取 `qmen_caiguan.json` 后，按以下步骤生成解读。所有数据已由脚本计算完成，SKILL 只负责用自然语言组织和解读数据，不做额外推算。

### 第一步：月令总论

**读取**：`yuegling` 对象

**字段**：
- `yuegling.branch` — 月令地支
- `yuegling.wuxing` — 月令五行
- `yuegling.palaces[]` — 每个宫位的月令关系，含 `relation`（英文标记）和 `relation_cn`（中文含义，如"月令克，大亏，量小。最差"）

**解读要求**：
- 说明月令地支和五行代表的大环境能量方向
- 遍历 `palaces[]`，按 relation 分组归纳：哪些宫被月令生助（sheng/tong），哪些宫被月令克制（ke_target）
- 给出"顺天时"的总体方向——优先选择被月令生助的宫位方向

### 第二步：财富七要害逐项诊断

**读取**：`caifu_yaohai` 数组

**每项字段**：
- `name` — 要害名称
- `desc` — 含义描述
- `type` — `stem`/`gate`/`deity`/`special`（special 类型引用对应专题数据）
- `palace` — 所在宫位号
- `palace_info` — 宫中星门神天盘地盘详情
- `liuhai` — 六害列表（逗号分隔）
- `liuhai_count` — 六害数量
- `yuegling_relation_cn` — 月令关系中文描述

**解读要求**：
- 逐项报告：名称、所在宫位、六害状态、月令关系
- 六害越多越严重，两个以上是危险信号
- `type=special` 的要害（月令、行业、干财）引用对应专题数据（`yuegling`、`hangye`、`gan_cai`）解读

**七要害解读要点**：

| 要害 | name | 正面含义 | 六害影响 |
|------|------|---------|---------|
| 戊 | 戊 | 本钱安稳 | 降薪、资产贬值、亏本 |
| 生门 | 生门 | 利润增长 | 回报不及预期、盈利有问题 |
| 六合 | 六合 | 合作顺利 | 人际关系网有麻烦、融资不顺 |
| 月令 | 月令 | 天时有利 | （special，从yuegling总论中解读） |
| 行业 | 行业 | 渠道适配 | （special，从hangye中解读） |
| 时干 | 时干 | 时机正确，正确的时间做正确的事 | 客户支持弱、员工支持弱、市场需求差、前景预期不好 |
| 干财 | 干财 | 资源可控 | （special，从gan_cai中解读） |

### 第三步：事业七要害逐项诊断

**读取**：`shiye_yaohai` 数组

与财富同样的字段结构和解读方法：

| 要害 | name | 正面含义 | 六害影响 |
|------|------|---------|---------|
| 开门 | 开门 | 公司运行好 | 氛围差、人际复杂、君子不立危墙 |
| 景门 | 景门 | 形象好业绩好 | 形象受损、成果不被认可 |
| 玄武 | 玄武 | 无小人 | 有阴险小人暗算，六害叠加尤为严重 |
| 庚/白虎 | 庚 | 无大压力 | 指标高、强敌、精神紧张 |
| 行业 | 行业 | 岗位适配 | （special，从hangye中解读） |
| 符使 | 符使 | 上级支持 | （special，从fushi中解读） |
| 诸天干 | 诸天干 | 人事关系好 | （special，从tiangan_roles中解读。天干有问题就是阻碍的根源。年干=大老板(最有权，不直接参与工作)，月干=同事，时干=下属） |

**玄武特别说明**：
- 玄武代表暗中力量，影响本宫和对宫
- 六害同宫时必须提醒用户警惕
- 玄武空亡或入墓尤其要小心
- 如果用户不从事玄学/哲学，玄武基本代表小人

**庚/白虎特别说明**：
- 庚和白虎本身就是问题来源
- 它们影响本宫和对宫
- 六害叠加（如击刑+门迫）情况更糟
- 但也可以是搏斗致富的渠道（笔记原文："唯一的求财渠道是拼命斗争"）

### 第四步：干财分析

**读取**：`gan_cai` 对象

**字段**：
- `ri_gan` / `nian_gan` — 日干/年干
- `ri_gan_cai` / `nian_gan_cai` — 对应的财干
- `stems[]` — 每个财干的状态：`stem`（天干）、`palace`（宫位）、`palace_info`（含 `state` 十二长生状态）、`liuhai`、`wuhe_alt`（五合替代）

**解读要求**：
- 列出日干财和生年财的天干
- 逐个报告在局中的宫位和六害状态
- 如果 `palace_info.state` 是"墓"，说明资源被困难以调动
- 如果 `wuhe_alt` 非空，说明该天干不在盘上，用五合替代
- 给出干财安全与否的判断

### 第五步：行业分析

**读取**：`hangye` 对象

**字段**：
- `palace` — 戊所在宫位
- `star` / `gate` / `deity` / `tian_gan` — 宫内符号
- `matched[]` — 脚本匹配到的行业列表
- `star_career` — 星的化气阵行业方向

**解读要求**：
- 说明戊落在哪个宫位，列出宫内符号
- 报告脚本匹配到的行业方向
- 如有 `star_career`，一并解读
- 结合月令关系说明行业难易程度

### 第六步："踩一捧一"综合建议

**读取**：`summary` 对象

**字段**：
- `caifu_total_liuhai` — 财富总六害数
- `shiye_total_liuhai` — 事业总六害数
- `caiyipengyi` — 脚本给出的踩捧建议

**解读要求**：
- 报告财富和事业的总六害数对比
- 引用 `caiyipengyi` 建议
- 结合前面各要害的具体情况，用日常语言展开踩捧建议
- 给出化解方向（参考各要害的六害类型）：
   - 入墓 → 资源被困，换时机或方位
   - 空亡 → 事情不实，不宜投入过多
   - 庚/白虎 → 正面硬刚或绕开
   - 门迫 → 换通道
   - 击刑 → 法律纠纷风险
- 参考月令生助的宫位方向给出时机和方位建议

## 追问处理

### 追问某个要害
用户问"戊的问题怎么解决"或"生门六害这么多怎么办"：
- 找到该要害在 JSON 中的完整数据（`palace_info`、`liuhai`、`yuegling_relation_cn`）
- 结合同宫的星门神组合解读具体状况
- 化解方向参考六害类型：换方位、换时间、换渠道

### 追问行业方向
用户问"我适合做什么行业"或"这个行业怎么样"：
- 行业已由脚本从戊所在宫自动推算，参考 `hangye` 数据中的匹配结果
- 结合月令关系分析该行业方向在当前大环境下的难易程度

### 需要布阵化解
如果用户想要灭象+布阵，引导使用 `qmen_huaqizhen` 技能：
> 如果你需要具体的布阵方案（在哪个方位摆什么物品来化解六害），可以使用化气阵技能。需要额外提供家人出生年份（可选，用于布阵保护）。
